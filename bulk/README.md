# 過去分の一括投稿（CSV一括インポート）

過去にためた写真を、**訪問日ベース**の日付でまとめて記事化するための仕組みです。
一度に全部でも、何回かに分けてでもOK（途中で止めても、続きから再開できます）。

## 用意するもの（Googleドライブ）

Googleドライブ（nozawa@beyonds.ai）の **マイドライブ直下に `BabaBlog-Bulk` フォルダ** を作り、その中に：

1. **`posts.csv`** … 投稿一覧（このフォルダの `posts.csv` がテンプレート）
2. **写真ファイル** … CSVの `photos` 列に書いたファイル名のとおりに置く（同じ `BabaBlog-Bulk` 内）

## CSV の列

| 列 | 必須 | 内容 | 例 |
|---|---|---|---|
| `visit_date` | ◯ | **訪問日**（この日付で記事が公開される） | `2024-05-03` |
| `category` | ◯ | `ごはん` / `子育て` / `旅` | `ごはん` |
| `shop_name` | ◯ | 店名・施設名・場所名 | `キッチンオトボケ` |
| `location` |   | 市区町村など | `東京都新宿区馬場下町` |
| `memo` |   | 感想・ひとことメモ（あると記事が良くなる） | `大盛りで満足。子連れは時間帯に注意。` |
| `photos` |   | 写真のファイル名。複数は **`;`（セミコロン）区切り**。無ければ空でOK（プレースホルダー＋地図になる） | `otoboke01.jpg;otoboke02.jpg` |

> Excel／Googleスプレッドシートで編集 →「CSV形式でエクスポート」→ `posts.csv` としてアップロードが簡単です。

## 処理の流れ

1. CSVと写真を `BabaBlog-Bulk` に置く
2. 一括インポートを実行（Claudeが処理）
3. 各行について：事実確認 → 訪問日の日付で記事生成 → 画像配置 → 公開
4. **1回あたり最大15件**ずつ処理（量が多くてもOK、複数回に分けて消化）

## 安全設計

- **重複なし**：記事のURL（slug）は `訪問日 + 店名` で決まり、すでに同じ記事があれば**スキップ**します。
  → 途中で止めても、もう一度実行すれば**続きから**進みます。何度実行しても二重投稿になりません。
- 写真が無い行は、カテゴリ別プレースホルダー画像＋Googleマップ埋め込みで記事化します。
